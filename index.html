What is GitHub?

Git is the engine.
GitHub is the road.

Git is a tool installed on your computer that tracks every change you make to your files — like an unlimited undo history for your code. It works completely offline.

GitHub is the website that hosts your Git projects online so you can access them anywhere, share them with teammates, and collaborate without emailing files back and forth.

Together they are how every professional development team in the world manages code.

Git GitHub

Core concepts

4 things that make everything click

01
Repository (Repo)
A smart project folder that stores all your files and remembers every single change ever made inside it. Think of it as a folder with a perfect memory.
github.com/username/repo-name
02
Commit
A saved snapshot of your work at a specific point in time. Every commit has a message explaining what changed. This is your permanent, searchable record.
git commit -m "your message"
03
Branch
A separate copy of your code where you can build a new feature or fix a bug without touching the main working version. Merge it back when you are done.
git checkout -b feature/name
04
Push & Pull
Push sends your commits from your computer up to GitHub. Pull brings the latest changes from GitHub down to your machine. This is how teams stay in sync.
git push / git pull
VS Code setup

Connect GitHub
to VS Code

VS Code has Git built in. Once connected to GitHub, you can commit, push and pull without ever leaving your editor.

Extension tip — Install GitHub Pull Requests and Issues from the VS Code marketplace to review and merge PRs without opening a browser.
1
Install Git
Download from git-scm.com and install with default settings. VS Code detects it automatically.
2
Set your identity
Open terminal with Ctrl + ` and run:
git config --global user.name "Your Name" git config --global user.email "you@email.com"
3
Sign in to GitHub
Click the Accounts icon bottom-left → Sign in with GitHub → Authorise in browser. One time only.
4
Clone a repository
Ctrl+Shift+P → Git: Clone → paste your repo URL.
git clone https://github.com/username/repo.git
5
Commit and push
Use the Source Control panel (Ctrl+Shift+G) or the terminal:
git add . && git commit -m "message" && git push
Writing good commits

Your commit history is a diary.
Make it readable.

Bad commits
fixed stuff
update
wip
changes
done finally
Good commits
feat: add dark mode toggle to settings
fix: resolve navbar collapse on mobile
style: reformat CSS to follow BEM naming
refactor: extract Button into own component
docs: update README with setup instructions
Format: type + colon + short description
feat fix style refactor docs chore test
Commands

The commands you will use every day

Setup
git config --global user.name "Name"Set your display name
git config --global user.email "email"Set your email
git clone <url>Download a repo to your machine
git initStart a new repo in a folder
Daily workflow
git statusSee what has changed
git add .Stage all changes
git commit -m "message"Save a snapshot
git push origin mainSend commits to GitHub
git pull origin mainGet latest from GitHub
Branches
git checkout -b feature/nameCreate and switch to a branch
git checkout mainSwitch back to main
git branchList all branches
git merge feature/nameMerge a branch into current
git branch -d feature/nameDelete branch after merge
Fixing things
git log --onelineSee commit history
git diffSee unstaged changes
git restore filenameUndo changes to a file
git stashSave work temporarily
git stash popBring stashed work back
.gitignore essentials
node_modules/Never commit dependencies
.envNever commit secrets or API keys
dist/ and build/Never commit build outputs
.DS_StoreMac system files — exclude them
Mistakes to avoid
git push --forceNever on shared branches
committing to mainAlways branch and use PRs
committing secretsSet up .gitignore before first commit
GitHub as your portfolio

Your GitHub profile is your live CV

Recruiters and hiring managers look at it. Here is what they actually pay attention to and how to make your profile stand out.

01
Contribution graph
The green squares showing daily activity. Consistent contributions over months signal a developer who codes regularly — not just before applying for jobs.
02
Pinned repositories
The 6 repos you feature at the top of your profile. Choose your best, most complete work — real projects with clear purpose and good documentation.
03
Profile README
A special README.md that appears at the top of your profile as a personal homepage. Introduce yourself, list your stack, and show what you are building.
04
Project quality over quantity
Fewer strong projects beat dozens of abandoned ones. Add a live demo link, screenshots in the README, and a clean commit history to every featured project.
